# Backend Implementation for PD Status

## MongoDB Schema Update

Add `pdStatus` and `pdDate` fields to your application schema:

```javascript
const applicationSchema = new mongoose.Schema({
  // ... existing fields ...
  
  status: {
    type: String,
    enum: ["Login", "PD", "Sanction", "Disbursed", "Part Disbursed", "Re-Login", "Rejected", "Withdraw", "Hold"],
    required: true
  },
  
  pdStatus: {
    type: String,
    enum: ["Yes", "No", ""],
    default: ""
  },
  
  pdDate: {
    type: Date,
    default: null
  },
  
  reloginReason: {
    type: String,
    default: ""
  },
  
  // ... other fields ...
});
```

## API Route Updates

### POST /api/applications (Create)

Ensure the route accepts `pdRemark`:

```javascript
router.post("/api/applications", async (req, res) => {
  try {
    const applicationData = {
      ...req.body,
      pdStatus: req.body.status === "PD" ? req.body.pdStatus || "" : "",
      pdDate: req.body.status === "PD" ? req.body.pdDate || null : null,
    };
    
    const newApplication = new Application(applicationData);
    await newApplication.save();
    
    res.status(201).json(newApplication);
  } catch (err) {
    console.error("Error creating application:", err);
    res.status(500).json({ error: "Failed to create application" });
  }
});
```

### PATCH /api/applications/:id (Update)

Ensure the route accepts `pdRemark`:

```javascript
router.patch("/api/applications/:id", async (req, res) => {
  try {
    const updateData = {
      ...req.body,
      pdStatus: req.body.status === "PD" ? req.body.pdStatus || "" : "",
      pdDate: req.body.status === "PD" ? req.body.pdDate || null : null,
    };
    
    const updatedApp = await Application.findByIdAndUpdate(
      req.params.id,
      updateData,
      { new: true, runValidators: true }
    );
    
    if (!updatedApp) {
      return res.status(404).json({ error: "Application not found" });
    }
    
    res.json(updatedApp);
  } catch (err) {
    console.error("Error updating application:", err);
    res.status(500).json({ error: "Failed to update application" });
  }
});
```

## Validation (Optional)

Add validation to ensure PD status and remark are provided when status is PD:

```javascript
applicationSchema.pre("save", function(next) {
  if (this.status === "PD") {
    if (!this.pdStatus || this.pdStatus.trim() === "") {
      return next(new Error("PD Status (Yes/No) is required when status is PD"));
    }
    if (!this.pdDate) {
      return next(new Error("PD Date is required when status is PD"));
    }
  }
  next();
});
```

## Migration Script (If Needed)

If you have existing applications, run this migration to add the pdRemark field:

```javascript
// migration.js
const Application = require('./models/Application');

async function migratePdFields() {
  try {
    await Application.updateMany(
      { 
        $or: [
          { pdStatus: { $exists: false } },
          { pdDate: { $exists: false } }
        ]
      },
      { 
        $set: { 
          pdStatus: "",
          pdDate: null
        } 
      }
    );
    
    console.log("✅ Migration completed: pdStatus and pdDate fields added to all applications");
  } catch (err) {
    console.error("❌ Migration failed:", err);
  }
}

migratePdFields();
```

## Frontend Changes Made

### 1. Added "PD" to Status Radio Buttons

The status options now include:
- Login
- **PD** (NEW)
- Sanction
- Disbursed
- Part Disbursed
- Re-Login
- Rejected
- Withdraw
- Hold

### 2. Added PD Status and Date Fields

When user selects "PD" status, two fields appear:

1. **PD Status Radio Button**:
   - Options: Yes / No
   - Positioned above PD Date
   - Stores user's selection

2. **PD Date Input**:
   - Type: Date picker
   - Required field (must be filled when PD is selected)
   - Stores date in YYYY-MM-DD format
   - Displays as DD-MM-YYYY in application list

### 3. Updated Form Data

Added `pdStatus: ""` and `pdDate: ""` to:
- `initialFormData` object
- `isFieldDisabled` function (both editable even after approval)
- `handleChange` editable fields list
- `handleEdit` function (loads existing PD status and date)
- `handleSubmit` function (only sends pdStatus and pdDate when status is "PD")

### 4. Display in Application List

When an application has status "PD", it displays:
```
PD Status: Yes
PD Date: 20-02-2026
```

Styled with:
- Orange text color (#ff9800)
- Light orange background (#fff3e0)
- Padding and border radius for visibility
- Two lines: first shows Yes/No status, second shows formatted date
- Date formatted as DD-MM-YYYY for Indian format
